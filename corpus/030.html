    IS 147 Introduction to Computer Programming Project

    Project Guidelines

    1. Create a complete application that includes all the project requirements.
    2. A program must have a complete input process and output.
    3. A program must be fully tested and should be error free.
    4. You should work as a team and each team member should contribute to the project.
    5. Your project source code must be accessible through GitHub.
    6. The team should work collaboratively to resolve any conflicts or disagreements during application development.
    7. All the requirements should be clearly defined and assigned to each team member through Github project application
    8. It is your team responsibility to track the progress of your application through Github project.

    Grading Process

    Project must be presented by all team members in class.

    Project will be graded by the instructor based on its completeness and presentation.
